Welcome To Angular List Tutorial Documentation Layout, CSS and navigation
The index.html is the boot page for the app, it loads partial views using routing (ex: this is home.html view) which is the default view
The menu on the top loads List/Manage View and add-item view.
You may notice the menu text disappears on small screens.
The app use angular material and theming configured in app.config.js
Use app.css to apply custom styles.
Functionality, Code, structure
App Structure: Applied the guidelines from https://github.com/johnpapa/angular-styleguide/blob/master/a1/README.md
- (Exception: For ControllerAs object I use short code for the controller like "lc" for ListController instead of vm for everything)
- Naming in format feature.type.js
- Files arranged per feature (ex: List files including managing and adding are under List folder)
- Scripts and vendors are under folder "scripts"
- Static Contents under folder "Content" (use app.css to apply custom css)
- Services/Factories and common files are under folder "common"

 Common Services and Factories
- app.commong.js: Define routing and theming
- data.service.js: handles database savings and getting data.
- Logger.service.js: logs events and errors
- navigate.service.js: handles navigation to another views using routing

 Data Service
- Data Service use localStorage as database
- Data Operation always use promises as if it is dealing with back end database
- All data is array based the object structure is Name/Descriptio/ID/Order
- Order number field is how we change the order of the items
- ID field is unique identifier created when creating new item in the array

 Deployment
- The JS and CSS are validated, minified and combined using Grunt (cssmin, Jshint, Uglify)
- Use the folder "Build" to publish the app
